# Autonomous Decomposition-Swarm Schema Name Map

Purpose: prevent drift between specification names, existing SQLite scaffolding, and final runtime names. Session 1 owns this map. Each worker session must add or update rows for its own persistence surfaces before implementation writes records.

Status values: `canonical`, `alias_to_existing`, `needs_new_migration`, `external_artifact_contract`, `unresolved`

## Naming Rules

1. Do not create two tables for the same logical slice.
2. If an existing table is semantically correct, keep it and document the spec alias.
3. If the spec name is clearer and the existing table is temporary/scaffold-only, migrate toward the spec name.
4. Runtime code imports names through a schema constant or repository helper, not ad hoc string literals.
5. Every `MIG-*` runtime surface must resolve to exactly one final table, view, or external artifact contract before runtime integration.
