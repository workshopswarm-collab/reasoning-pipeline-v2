# Autonomous Decomposition-Swarm Live-Cutover Blocker Matrix

Purpose: translate the explicit Section 18 live-cutover blockers into an owner/test matrix. `v2_live_cutover` cannot proceed unless every blocker row is `passed`, `not_applicable_with_reason`, or explicitly waived by the owning session and Session 1.

Status values: `not_started`, `in_progress`, `blocked`, `passed`, `not_applicable_with_reason`, `waived`

## Global Rules

- A blocker waiver must include owner, reason, expiry, affected cases, and compensating control.
- A blocker cannot be waived when it would create a second live forecast authority.
- Runtime integration must record blocker evidence in the shared inventory or fixture result.
- This matrix is stricter than the feature inventory: a feature can be `ready_for_integration` while a blocker remains open for live cutover.
